# BugFlow - 轻量级SaaS化缺陷管理与协作平台

> 一个基于 Django 构建的现代化、功能全面的项目协作与缺陷追踪平台。

---

## 🌟 项目简介 / プロジェクト概要

BugFlow 是一款专为开发团队设计的轻量级SaaS协作平台。它集成了项目管理、任务/Bug追踪、在线文档编辑与实时数据统计，旨在提升团队协作效率，保证项目质量与进度。

BugFlowは、開発チーム向けに設計された軽量SaaSコラボレーションプラットフォームです。プロジェクト管理、タスク/Bug追跡、オンラインドキュメント編集、リアルタイムデータ統計を統合し、チームのコラボレーション効率とプロジェクトの品質・進捗管理を向上させることを目的としています。

## 🚀 核心功能 / コア機能

### 👥 用户系统 / ユーザーシステム
- **安全认证：** 支持用户注册、登录，集成SMTP邮件验证码与Pillow图形验证码双重校验。
  - **セキュア認証：** ユーザー登録、ログインをサポート。SMTPメール認証コードとPillowグラフィカル認証コードによる二重検証を統合。
- **成员邀请：** 通过邮件邀请团队成员加入特定项目。
  - **メンバー招待：** メールを通じてチームメンバーを特定のプロジェクトに招待。

### 📦 项目管理 / プロジェクト管理
- **全生命周期管理：** 创建、编辑项目，并需在管理界面进行项目名确认后方可删除，防止误操作。
  - **ライフサイクル管理：** プロジェクトの作成、編集、および管理画面でのプロジェクト名確認による削除をサポートし、誤操作を防止。

### 🐞 问题追踪 / 課題追跡
- **精细化任务管理：** 创建问题时，可指定优先级、类型（Bug/功能/任务）、状态，并指派给管理者。
  - **詳細なタスク管理：** 課題作成時に、優先度、種別（Bug/機能/タスク）、状態を指定し、管理者に割り当て可能。
- **动态讨论：** 支持问题回复讨论，所有动态（状态变更、内容修改、讨论）均自动记录并更新。
  - **動的ディスカッション：** 課題への返信とディスカッションをサポート。全ての動的（状態変更、内容修正、ディスカッション）は自動的に記録・更新されます。

### 📎 文件与文档 / ファイルとドキュメント
- **云文件服务：** 集成腾讯云COS对象存储，实现高效稳定的文件上传与下载。
  - **クラウドファイルサービス：** Tencent Cloud COSオブジェクトストレージと統合し、効率的で安定したファイルのアップロードとダウンロードを実現。
- **在线文档：** 集成editor.md插件，提供强大的Markdown富文本编辑与实时预览功能。
  - **オンラインドキュメント：** editor.mdプラグインを統合し、強力なMarkdownリッチテキスト編集とリアルタイムプレビュー機能を提供。

### 📊 数据可视化 / データ可視化
- **智能数据面板：** 集成ECharts，动态展示问题分布、人员负载、处理进度等关键项目指标。
  - **インテリジェントデータダッシュボード：** EChartsを統合し、課題分布、担当者負荷、処理進捗などの主要なプロジェクト指標を動的に表示。

## 🛠 技术栈 / テクノロジースタック

### 后端 / バックエンド
- **Python 3.x**
- **Django 3.2**
- **Pillow** (图形验证码生成 / グラフィカル認証コード生成)

### 前端 / フロントエンド
- **HTML5, CSS3, JavaScript (ES6+)**
- **Bootstrap** (响应式UI框架 / レスポンシブUIフレームワーク)
- **jQuery** (DOM操作与Ajax / DOM操作とAjax)
- **ECharts** (数据可视化 / データ可視化)
- **editor.md** (Markdown编辑器 / Markdownエディタ)

### 数据库与缓存 / データベースとキャッシュ
- **MySQL** (主数据库 / メインデータベース)
- **Redis** (验证码缓存与会话管理 / 認証コードキャッシュとセッション管理)

### 云服务与第三方 / クラウドサービスとサードパーティ
- **Tencent Cloud COS** (对象存储服务 / オブジェクトストレージサービス)
- **SMTP** (邮件发送服务 / メール送信サービス)

## ✨ 技术亮点与难点攻克 / 技術的ハイライトと難点攻略

1.  **架构现代化升级：** 项目初期基于陈旧的Django 1.11教程，我主导并成功将其升级至**Django 3.2**。在此过程中，尝试并解决了版本差异带来的功能以及写法的变更、redis配置使用流程及兼容性等问题。
    - **アーキテクチャの近代化アップグレード：** プロジェクト初期は旧式のDjango 1.11チュートリアルに基づいていたが、**Django 3.2 LTS** へのアップグレードを主導して成功させた。この過程で、バージョン差異によるAPI変更、ミドルウェア実行フロー、セキュリティポリシー設定などの互換性問題を深く理解し解決した。

2.  **核心服务集成与替换：** 摒弃了教程中个人开发者难以申请的腾讯云短信服务，独自研究并集成**SMTP邮件服务**，实现了用户注册验证流程。同时集成**腾讯云COS**，为平台提供了企业级的文件存储解决方案。
    - **コアサービス統合と置換：** 個人開発者には申請が困難なチュートリアル内のTencent Cloud SMSサービスを廃し、**SMTPメールサービス**の独自調査・統合によりユーザー登録認証フローを実現。同時に、**Tencent Cloud COS**の統合に成功し、プラットフォームにエンタープライズ級のファイルストレージソリューションを提供。

3.  **安全性与时效性设计：** 采用**Redis**作为高速缓存数据库，存储邮件与图形验证码，并设置自动过期时间，兼顾了系统安全与用户体验。登录环节加入**Pillow生成的图形验证码**，有效防止恶意爆破。
    - **セキュリティと時効性の設計：** **Redis**を高速キャッシュデータベースとして採用し、メールとグラフィカル認証コードを保存し、自動 expiration を設定、システム安全性とユーザーエクスペリエンスを両立。ログイン段階に**Pillowで生成したグラフィカル認証コード**を追加し、悪質な攻撃を効果的に防止。
